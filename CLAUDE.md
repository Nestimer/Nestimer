# CLAUDE.md

Guidance for future Claude Code sessions working on this repo.

## What this is

Self-hosted parental controls for macOS. 4 components:
- **API** (`api/`) — FastAPI + SQLAlchemy async + SQLite
- **macOS Agent** (`macos-agent/`) — Swift menu bar app that locks the screen
- **Parent App** (`ParentApp/`) — SwiftUI, iOS 17+ / macOS 14+
- **Web Dashboard** (`web-dashboard/`) — React + Vite

Deployed via Docker Compose (api + nginx serving web build).

## Build commands

All builds in this project need the `CODE_SIGN_*` overrides below to work locally without a paid Apple Developer account.

```bash
# Agent (Debug — auto-enables dev mode)
xcodebuild -project macos-agent/UsageTimeAgent.xcodeproj \
  -scheme NesTimerAgent -configuration Debug \
  CODE_SIGN_ALLOW_ENTITLEMENTS_MODIFICATION=YES build

# Agent (Release — production, root install, no dev safeguards)
xcodebuild -project macos-agent/UsageTimeAgent.xcodeproj \
  -scheme NesTimerAgent -configuration Release \
  CODE_SIGN_ALLOW_ENTITLEMENTS_MODIFICATION=YES \
  CODE_SIGN_IDENTITY="-" CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=NO build

# Parent app (macOS)
xcodebuild -project ParentApp/UsageTimeControl.xcodeproj \
  -scheme UsageTimeControl -configuration Debug \
  -destination 'platform=macOS' \
  CODE_SIGN_ALLOW_ENTITLEMENTS_MODIFICATION=YES build
```

Built apps land in `~/Library/Developer/Xcode/DerivedData/<project>-*/Build/Products/<config>/*.app`.
Release builds get copied to `dist/` for distribution.

## Key architectural decisions

### Agent lifecycle (Release)
1. User double-clicks `UsageTimeAgent.app` from anywhere
2. Setup dialog asks for `http://server:8000|TOKEN` setup string → saves in UserDefaults
3. `SystemInstaller.promptAndInstallIfNeeded()` asks for admin password via `NSAppleScript` → copies app to `/Applications/` (root) and installs `/Library/LaunchDaemons/com.usagetime.agent-watchdog.plist`
4. The daemon watchdog runs every 15 s as root and relaunches the agent as the console user via `launchctl asuser`
5. Debug builds skip `SystemInstaller` entirely

### TOTP
- 160-bit hex shared secret per device, generated by API on device creation
- HMAC-SHA1, **300 s step** (5 min, not 30 s), 6 digits, ±1 step tolerance
- Python (`api/app/totp.py`), Swift (both agent and parent app have their own `TOTPGenerator.swift`), pure JS (`web-dashboard/src/utils/totp.js` — no Web Crypto API because HTTP serves don't have `crypto.subtle`)
- All three implementations must stay in sync
- TOTP-granted temp unlock = 30 min during which ALL checks are bypassed (downtime + limit) AND `performTick()` pauses counting

### Policy evaluation order (`PolicyEnforcer.evaluate`)
```
1. Active scheduled activity?  → unlock, return (also pauses counting)
2. Temp TOTP unlock active?    → unlock, return (also pauses counting)
3. Downtime window?            → lock
4. remaining < 1 minute?       → lock
5. cross warning thresholds    → notify
6. otherwise                   → unlock
```
Everything short-circuits. If you add a new rule, pick the right priority.

### Lock screen window (`LockScreenWindow.swift`)
- `NSWindow` subclass `KeyableWindow` (borderless windows can't be key by default → no keyboard input)
- `isReleasedWhenClosed = false` (otherwise double-free crash with array holding the reference)
- `LockReason` enum is **`indirect`** (has a case that wraps itself)
- `findTextField(in:)` recursively walks NSView hierarchy to force first-responder on the SwiftUI `TextField` after window becomes key
- TOTP input is **always visible** (never hide it behind a button — keyboard focus is fragile and the user gets stuck)

### Datetime handling (API)
- All writes use `datetime.now(timezone.utc)` (never `utcnow()`)
- Columns use `DateTime(timezone=True)`
- Pydantic schemas have `field_serializer` that coerces naive datetimes (from legacy rows) to UTC before ISO-serializing
- Swift side tries `.withFractionalSeconds` first, falls back to `.withInternetDateTime`
- Web side: format includes `+00:00` offset, so `new Date()` works correctly

### Migrations
- No Alembic — migrations live in `api/app/database.py::init_db()`
- For added columns: `PRAGMA table_info(table)` → check if missing → `ALTER TABLE … ADD COLUMN …`
- For new tables: `Base.metadata.create_all` handles it
- Only additive migrations. Don't drop/rename.

### Sync intervals
- Debug: tick 5 s, sync 10 s
- Release: tick 30 s, sync 20 s normally, **5 s while locked** (critical — we don't want the user stuck waiting for unlock)
- The sync timer fires every 5 s and only does network work on every Nth tick (when not locked)

## Project conventions

- **No Russian text** in UI, logs, or docstrings. All English. The repo was translated; keep it that way.
- **Emoji usage** — only in READMEs / commit messages if helpful. Not in code.
- **Git commit style** — conventional commits (`feat:`, `fix:`, `refactor:`, `test:`). Co-Authored-By footer for Claude commits.
- **Pre-built binaries** go in `dist/` (gitignored).
- **Icons** are generated by `gen_icons.py` (Pillow). Three variants: controller (parent app), agent (menu bar), app (generic). Preview at `icons/preview.png`.
- **Login**: devices and parent accounts are isolated. No roles/permissions beyond "owner of device".

## Testing & safety

- **Never run Release installer** on your main Mac without a recovery plan
- Dev mode has emergency hotkey **Ctrl+Opt+Cmd+U** + auto-unlock after 10 s
- If stuck: Safe Mode (Shift at boot) or Recovery Mode (Cmd+R or hold Power on Apple Silicon)
- API has ~80 tests covering auth, devices, policy, usage, TOTP, activities, e2e flow
- `pytest api/tests/` must pass before any commit that touches the API

## Known sharp edges

1. **Xcode project files** (`*.xcodeproj/project.pbxproj`) — when adding a new Swift file, Xcode updates the pbxproj automatically. CLI edits (via `Write`/`Edit`) mean you have to manually add `PBXBuildFile`, `PBXFileReference`, add to a group, and add to `PBXSourcesBuildPhase`. Skipping any step = `cannot find 'X' in scope`.
2. **macOS deployment targets** — Parent app uses 14+/iOS 17+ because of `ContentUnavailableView` and two-param `onChange`. Agent uses 13+ so it can run on older machines; availability-check any 14-only API.
3. **ATS** — both agent and parent app need `NSAllowsArbitraryLoads = true` to talk to HTTP APIs (agent via `Info.plist`, parent app via custom `Info.plist` + `INFOPLIST_FILE` build setting).
4. **Web Crypto API** — not available over plain HTTP, so `totp.js` ships its own SHA-1 + HMAC-SHA1. Don't re-introduce `crypto.subtle` unless you can guarantee HTTPS.
5. **FastAPI 422 errors** on the web dashboard — `detail` is an **array** of validation errors, not a string. `web-dashboard/src/services/api.js` has a formatter; don't revert it.
