# CLAUDE.md

Guidance for future Claude Code sessions working on this repo.

## What this is

**NesTimer** — parental controls for macOS. 4 components:
- **API** (`api/`) — FastAPI + SQLAlchemy async + PostgreSQL
- **macOS Agent** (`macos-agent/`) — Swift menu bar app that locks the screen
- **Parent App** (`ParentApp/`) — SwiftUI, iOS 17+ / macOS 14+
- **Web Dashboard** (`web-dashboard/`) — React + Vite

Production: `https://nestimer.com` (marketing), `https://my.nestimer.com` (dashboard + API).
Deployed via Docker Compose (PostgreSQL + API + nginx) on a 2 vCPU / 8 GB server.

```bash

xcodebuild -project macos-agent/UsageTimeAgent.xcodeproj \
  -scheme UsageTimeAgent -configuration Debug \
  CODE_SIGN_ALLOW_ENTITLEMENTS_MODIFICATION=YES build

# Agent (Release — production, root install, no dev safeguards)

  -scheme UsageTimeAgent -configuration Release \
  CODE_SIGN_ALLOW_ENTITLEMENTS_MODIFICATION=YES \
  CODE_SIGN_IDENTITY="-" CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=NO build

# Parent app (macOS)
xcodebuild -project ParentApp/UsageTimeControl.xcodeproj \
  -scheme UsageTimeControl -configuration Debug \
  -destination 'platform=macOS' \
  CODE_SIGN_ALLOW_ENTITLEMENTS_MODIFICATION=YES build
```

Release builds go to `dist/`. Push updates via:
```bash
./push-agent-update.sh 134.209.8.62 <version>
# Auto-bumps MARKETING_VERSION, rebuilds, zips, uploads, strips quarantine
```

## Key architectural decisions

### Agent lifecycle (Release)
1. User double-clicks `UsageTimeAgent.app`
2. Setup dialog asks for `https://my.nestimer.com|TOKEN` → saves in UserDefaults
3. `SystemInstaller` asks for admin password → copies app to `/Applications/` (root), installs bundled `watchdog.sh` + `watchdog.plist` as LaunchDaemon
4. Watchdog runs every 15s as root, restarts agent if killed, checks for auto-updates every 5 min
5. Agent migrates old `http://134.209.8.62:8000` URLs to `https://my.nestimer.com` automatically
6. Debug builds skip `SystemInstaller` entirely

### TOTP
- 160-bit hex shared secret per device, generated by API on creation
- HMAC-SHA1, **300s step** (5 min), 6 digits, ±1 step tolerance
- **Security**: secret NOT sent in `/agent/config` (child could read it). Agent fetches once via `/agent/totp-secret`, stores in **Keychain** (not UserDefaults)
- Server-side verification via `/agent/verify-totp` (rate limited: 5 attempts per 5 min)
- Offline verification: agent uses Keychain-cached secret
- Python (`api/app/totp.py`), Swift (`TOTPGenerator.swift`), pure JS (`totp.js` — no Web Crypto API)
- All three implementations must stay in sync

### Policy evaluation order (`PolicyEnforcer.evaluate`)
```
1. Active scheduled activity?  → unlock, return (pauses counting)
2. Temp TOTP unlock active?    → unlock, return (pauses counting)
3. Downtime window?            → lock
4. remaining < 1 minute?       → lock
5. cross warning thresholds    → notify (AppleScript `display notification`)
6. otherwise                   → unlock
```

### Adaptive sync intervals
Agent adjusts sync frequency based on state:
- **Locked**: every 5s (parent might change policy)
- **< 5 min remaining**: every 10s
- **< 30 min remaining**: every 20s
- **Idle (> 30 min left)**: every 60s
- **During activity/TOTP unlock**: every 60s
- **Dev mode**: every 10s

Timer fires every 5s, checks `adaptiveSyncInterval()` vs `lastSyncTime`.

### Lock screen (`LockScreenWindow.swift`)
- `KeyableWindow` subclass (borderless NSWindow can't be key by default)
- `isReleasedWhenClosed = false` (prevents double-free crash)
- `LockReason` enum is `indirect` (recursive)
- TOTP field always visible with `@FocusState` + `onAppear { codeFieldFocused = true }`
- Auto-submit when 6 digits entered
- Rebuilds windows when screen count changes (monitor plug/unplug)

### Datetime handling
- All writes: `datetime.now(timezone.utc)` (never `utcnow()`)
- DB columns: `DateTime(timezone=True)`
- Agent sends `?date=YYYY-MM-DD` (local date) to `/agent/config` — API uses it for usage lookup AND weekday calculation (fixes timezone mismatch at day boundaries)
- Pydantic `field_serializer` coerces naive datetimes to UTC
- Swift: `ISO8601DateFormatter` with `.withFractionalSeconds` fallback

### Database
- **PostgreSQL 16** in Docker (was SQLite, migrated)
- Migrations in `api/app/database.py::init_db()` using SQLAlchemy `inspect` (dialect-agnostic)
- Tests still use SQLite in-memory (`TESTING=1` env disables rate limiter)

### Security
- `SECRET_KEY` — fail-fast if default value on startup
- CORS restricted to `my.nestimer.com` + localhost (not wildcard)
- Rate limiting: login 5/min, register 3/5min, TOTP verify 5/5min per device
- Agent API port 8000 localhost-only (all traffic through nginx HTTPS)
- DevMode only via `#if DEBUG` (child can't enable via UserDefaults)
- Docker API runs as non-root user
- TOTP secret in agent Keychain (not UserDefaults)
- Notifications via AppleScript (UNUserNotification silently fails for unsigned apps)

## Known sharp edges

1. **pbxproj editing** — adding Swift files requires manual `PBXBuildFile` + `PBXFileReference` + group + `PBXSourcesBuildPhase` entries. Miss one = `cannot find 'X' in scope`.
2. **Deployment targets** — Parent app: macOS 14+ / iOS 17+. Agent: macOS 13+ (availability-check 14-only APIs).
3. **ATS** — both apps need `NSAllowsArbitraryLoads = true` in Info.plist for HTTP fallback.
4. **Web Crypto API** — `totp.js` uses pure JS SHA-1 (no `crypto.subtle`). Don't change unless HTTPS is guaranteed.
5. **FastAPI 422** — `detail` is an array, not string. `api.js` has a formatter.
6. **Rate limiter** — disabled when `TESTING=1` env is set. CI sets this automatically.
7. **Watchdog script** — bundled inside `.app/Contents/Resources/`. `SystemInstaller` copies it to `/usr/local/libexec/`. If you change `watchdog.sh`, update BOTH `macos-agent/Watchdog/watchdog.sh` AND `macos-agent/UsageTimeAgent/watchdog.sh`.
8. **Agent URL migration** — `AgentConfig.migrateServerURL()` auto-converts `134.209.8.62:8000` to `my.nestimer.com`. Add new migrations there if server URL changes.
