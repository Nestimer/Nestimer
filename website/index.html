Why UsageTime Controller?

The problem

  • Apple Screen Time is easy to bypass
  • Commercial solutions cost $5-15/month per child
  • Cloud services collect your family's data
  • Most tools don't offer flexible per-day limits
  • No offline fallback when the internet goes down

The solution

  • Root-protected agent that can't be killed or quit
  • Completely free and open-source
  • Self-hosted — your data stays on your server
  • Per-day limits, weekday/weekend schedules
  • Offline unlock codes work without internet

Features

⏱

Screen Time Limits

Set a daily limit in minutes. Different limits for weekdays and weekends, or override each day individually. Only counts time when your child is actively using the Mac — sleep, lock screen, and idle time don't count.

☾

Downtime Schedule

Lock the computer on a schedule — for example, 10 PM to 8 AM on school nights, midnight to 9 AM on weekends. Separate weekday and weekend windows.

📚

Scheduled Activities

Create activity windows for classes, lessons, or practice. The computer unlocks automatically during these times, and the minutes don't count toward the daily limit. Configurable buffer time before and after.

🔒

Offline Unlock Codes

6-digit codes that change every 5 minutes. Works without internet. Read the code from your phone or the web dashboard, and your child types it on the lock screen for a 30-minute temporary unlock.

🛡

Tamper Protection

The agent installs as a root-protected system daemon. A watchdog process checks every 15 seconds and restarts it if killed. Cmd+Q is intercepted. Requires admin access to uninstall.

🌐

Web Dashboard

Manage everything from your browser. See today's usage, change limits, view history, and get unlock codes. Works on any device — phone, tablet, or computer.

📱

Native Parent App

SwiftUI app for iPhone, iPad, and Mac. Same controls as the web dashboard, optimized for Apple devices.

💫

Smart Tracking

Usage time is only counted when the screen is on, unlocked, and the user is active. If your child walks away for 5+ minutes, the timer pauses automatically.

🔐

Private & Self-Hosted

Deploy on any Linux server with one command. Your family's data stays on your hardware. No accounts with third-party services, no tracking, no ads.

How It Works

1

Deploy the server

Run docker compose up -d on any Linux VPS or home server. The API and web dashboard start automatically. Takes about 2 minutes.

2

Add a device

Open the web dashboard, create an account, and add your child's Mac. You'll get a setup string to paste into the agent.

3

Install the agent

Download the agent app on your child's Mac and paste the setup string. It will ask for the admin password once to install the system daemon.

4

Set limits

From the web dashboard or parent app, configure daily limits, downtime schedules, and activity windows. Changes sync to the agent within seconds.

Frequently Asked Questions

What is UsageTime Controller?

A free, open-source, self-hosted parental control system for macOS. It lets parents set screen time limits, downtime schedules, and manage their child's Mac remotely via a web dashboard or native app.

Is it really free?

Yes, completely free and open-source. You host the server yourself (on a VPS or home server), so there are no subscription fees. The only cost is your server hosting (a $5/month VPS is more than enough).

Can my child bypass the limits?

The agent installs as a root-protected LaunchDaemon with a watchdog process. It cannot be quit via Cmd+Q, and if the process is killed, the watchdog restarts it within 15 seconds. The child would need admin (sudo) access to uninstall it. If you set up your child as a Standard User on macOS, they cannot remove it.

What if the internet goes down?

All policies are enforced locally on the Mac, even without internet. Parents can generate offline unlock codes (TOTP) from the dashboard or app. The code changes every 5 minutes and grants a 30-minute temporary unlock.

How is this different from Apple Screen Time?

Apple Screen Time requires iCloud Family Sharing and is widely known to be easy for kids to bypass. UsageTime Controller is root-protected, self-hosted (your data stays private), supports offline unlock codes, and offers more flexible scheduling with per-day limits and activity windows.

What do I need to run it?

A server (any Linux VPS or home server) running Docker, and a Mac (macOS 13 or newer) for the child. Parents manage everything from a web browser or the native iOS/macOS app.

Does it track what websites my child visits?

No. UsageTime Controller only tracks total screen time (active use). It does not monitor browsing history, app usage, or any content. It's a time management tool, not a surveillance tool.

Can I use it for multiple children?

Yes. Add multiple devices to your account, each with its own limits, schedules, and activity windows.

How do I deploy the server?

Clone the repository, run docker compose up -d --build, and you're done. The API runs on port 8000 and the web dashboard on port 3080. Put Caddy or nginx in front for HTTPS.

Is it safe to run on my child's Mac?

The agent is designed to be non-intrusive. It runs silently in the menu bar, only showing a lock screen when limits are reached. In Debug mode, there's an emergency hotkey (Ctrl+Opt+Cmd+U) and auto-unlock after 10 seconds for testing.
